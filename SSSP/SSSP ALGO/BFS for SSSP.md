

## 1. 核心问题

**SSSP = Single-Source Shortest Path**

给定一个源点 `s`，求从 `s` 到图中每个点 `u` 的最短路径长度 `δ(s, u)`，以及对应的实际路径。

---

## 2. BFS 能解决哪类 SSSP？

BFS 可以在线性时间 **O(V+E)** 内解决以下特殊情况：

- **无权图** 的最短路
- **所有边权都相同** 的图的最短路

例如：
- 所有边权都为 `1`
- 所有边权都为某个相同正常数 `k`

这时，“最短路径”就等价于：

**从源点到目标点经过的边数最少。**

因为每条边代价相同，所以：

- 边数最少
- 总权重最小

这两个目标本质上是一样的。

---

## 3. 为什么 BFS 可以求无权图最短路？

BFS 的遍历方式是 **一层一层向外扩展**：

- 第 0 层：源点 `s`
- 第 1 层：与 `s` 相邻的点
- 第 2 层：距离 `s` 两条边的点
- 第 3 层：距离 `s` 三条边的点
- ...

所以，BFS **第一次访问到某个点时**，就已经保证：

**它是通过最少边数到达的。**

而在无权图里，最少边数路径就是最短路径。

因此：

**BFS 生成的 spanning tree 恰好就是无权图的 shortest path tree。**

---

## 4. 时间复杂度优势

- BFS：`O(V+E)`
- Bellman-Ford：`O(V×E)`

这里要注意：

- BFS 是 `+`
- Bellman-Ford 是 `×`

所以当图是无权图时，用 BFS 几乎是毫无疑问的选择。

---

## 5. 如何把普通 BFS 改成求 SSSP 的 BFS？

普通 BFS 通常只关心一个点是否访问过，所以会使用：

`visited[]`

而求最短路时，我们不仅要知道“访问过没有”，还要知道“距离是多少”，所以要改成：

`D[]`

其中：

- `D[u]` 表示从源点 `s` 到 `u` 的当前最短距离

---

## 6. 修改 1：visited 数组改成距离数组 D

普通 BFS：

    bool visited[V];

SSSP 版 BFS：

    int D[V];

初始化方式改为：

    for each vertex u:
        D[u] = INF
    D[s] = 0

其中：

- `INF` 可以取一个很大的数，比如 `1e9`
- 也可以用 `-1` 表示“未访问”
- **不能统一初始化成 0**，因为源点本来就满足 `D[s] = 0`

所以：

- 对所有 `u ≠ s`，设 `D[u] = INF`
- 对源点 `s`，设 `D[s] = 0`

---

## 7. 修改 2：访问条件改成检查 D[v]

普通 BFS 中常写：

    if (visited[v] == false) {
        visited[v] = true;
        ...
    }

改成最短路版本后：

    if (D[v] == INF) {
        D[v] = D[u] + 1;
        ...
    }

含义是：

- 如果 `v` 还没有被访问
- 那么从 `u` 走一步到 `v`
- 所以 `v` 的距离就是 `D[u] + 1`

---

## 8. BFS 求无权图最短路的伪代码

    BFS(s):
        for each vertex u in V:
            D[u] = INF
            p[u] = -1

        D[s] = 0
        create an empty queue q
        q.push(s)

        while q is not empty:
            u = q.front()
            q.pop()

            for each neighbor v of u:
                if D[v] == INF:
                    D[v] = D[u] + 1
                    p[v] = u
                    q.push(v)

其中：

- `D[v]`：从源点到 `v` 的最短距离
- `p[v]`：`v` 的父节点，用于恢复路径

---

## 9. 为什么 `D[v] = D[u] + 1`？

因为无权图中每条边长度都视为 `1`。

所以：

- 如果从 `s` 到 `u` 的最短距离是 `D[u]`
- 再从 `u` 走一条边到 `v`
- 那么到 `v` 的距离自然就是 `D[u] + 1`

而 BFS 是分层扩展的，第一次更新到 `v` 时，这个值一定已经是最小的。

---

## 10. 路径恢复

如果只要求最短距离，那么 `D[]` 就够了。

如果还要求具体路径，就需要维护父节点数组：

    p[v] = u

意思是：`v` 是从 `u` 扩展得到的。

恢复路径时，从终点不断往回跳：

    v -> p[v] -> p[p[v]] -> ... -> s

最后反转顺序即可得到从 `s` 到 `v` 的最短路径。

---

## 11. BFS 为什么不能解决一般带权图最短路？

因为 BFS 真正优化的是：

**经过的边数最少**

但一般带权图要求优化的是：

**路径总权重最小**

这两个目标不一定相同。

---

## 12. 反例

假设从 `s` 到 `t` 有两条路：

- 路径 A：1 条边，权重 = 100
- 路径 B：3 条边，权重 = 1 + 1 + 1 = 3

BFS 会选路径 A，原因是它边数更少。

但真正的最短路应该是路径 B，因为它总权重更小。

所以：

**BFS 只保证边数最少，不保证总权重最小。**
