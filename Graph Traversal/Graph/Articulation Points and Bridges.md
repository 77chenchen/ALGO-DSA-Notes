

## 1. 基本概念

### Articulation Point（割点）
在**无向图**中，如果删除某个点 `u` 以及和它相连的边后，图的连通分量数量增加，那么这个点就是 **Articulation Point（割点）**。

直观理解：
- 这个点像“交通枢纽”
- 一旦删掉，原本连通的图会被“断开”

---

### Bridge（桥）
在**无向图**中，如果删除某条边 `(u, v)` 后，图的连通分量数量增加，那么这条边就是 **Bridge（桥）**。

直观理解：
- 这条边像“唯一通道”
- 一旦删掉，两边就无法互相到达

---

## 2. 核心问题

割点和桥本质上都在问：

> 当前这个点 / 这条边，是否是某部分图连接到前面部分的唯一方式？

如果是唯一方式，那它就是关键结构。

---

## 3. DFS 解法的核心变量

我们通常用一次 DFS 来求，维护两个数组：

- `num[u]`：节点 `u` 第一次被 DFS 访问到的时间戳
- `low[u]`：从 `u` 出发，沿着若干条 DFS 树边，再加上**最多一条 back edge**，所能到达的最早祖先的 `num` 值

---

## 4. `low[u]` 的直观含义

`low[u]` 表示：

> 从 `u` 这棵子树出发，能不能“绕开父边”，回到更早的祖先。

- 如果能回去，说明这棵子树不是完全依赖父节点连接上层
- 如果回不去，说明它只能靠父节点或父边连接上层

所以：
- 回不去 ⇒ 父边可能是桥
- 回不去 ⇒ 父节点可能是割点

---

## 5. DFS 中遇到邻居时的两种情况

对于当前点 `u` 的邻居 `v`：

### 情况 1：`v` 还没访问过
说明 `(u, v)` 是 **tree edge（树边）**

先递归：

~~~text
DFS(v)
~~~

递归回来后更新：

~~~text
low[u] = min(low[u], low[v])
~~~

---

### 情况 2：`v` 已经访问过，并且 `v != parent[u]`
说明遇到了 **back edge（返祖边）**

更新：

~~~text
low[u] = min(low[u], num[v])
~~~

注意：
- 这是**无向图**
- 所以一定要排除父边
- 否则会把回到父节点那条边误判成 back edge

---

## 6. Bridge（桥）的判定条件

对于一条 DFS 树边 `(u, v)`：

如果：

~~~text
low[v] > num[u]
~~~

那么 `(u, v)` 是 **Bridge**

### 为什么是 `>`？
这说明：
- `v` 子树无法通过任何 back edge 回到 `u` 或 `u` 的祖先
- 所以 `v` 这部分只能通过 `(u, v)` 连到上面
- 删掉这条边后，图就断开了

---

## 7. Articulation Point（割点）的判定条件

### 情况 1：`u` 不是 DFS 根节点
如果存在某个 DFS 子节点 `v` 满足：

~~~text
low[v] >= num[u]
~~~

那么 `u` 是 **Articulation Point**

### 为什么是 `>=`？
这说明：
- `v` 子树无法绕过 `u` 去到 `u` 的祖先
- 即使 `low[v] == num[u]`，也只是能回到 `u` 自己，还是绕不过 `u`
- 所以删掉 `u` 后，`v` 子树会和上面断开

---

### 情况 2：`u` 是 DFS 根节点
如果根节点有 **两个或以上 DFS 子树**，那么它是 **Articulation Point**

即：

~~~text
rootChildren >= 2
~~~

### 为什么根节点特殊？
因为根节点没有父节点，所以不能直接套用 `low[v] >= num[u]`

根只要有两棵及以上 DFS 子树，就说明：
- 这些子树彼此之间只能通过根连接
- 删除根后，它们就会分开

---

## 8. 条件对比总结

| 结构 | 条件 |
|------|------|
| Bridge `(u, v)` | `low[v] > num[u]` |
| Articulation Point `u`（非根） | 存在子节点 `v` 使 `low[v] >= num[u]` |
| Articulation Point `root` | `rootChildren >= 2` |

---

## 9. 为什么桥是 `>`，割点是 `>=`

这是最关键也最容易混淆的地方。

### Bridge 用 `>`
桥关心的是：删掉边 `(u, v)` 后，`v` 子树还能不能连回去。

如果 `low[v] == num[u]`，说明 `v` 子树还能回到 `u`，虽然回不到 `u` 的祖先，但边 `(u, v)` 不一定是唯一连接方式，因此**不一定是桥**。

所以桥必须要求更严格：

~~~text
low[v] > num[u]
~~~

---

### Articulation Point 用 `>=`
割点关心的是：删掉点 `u` 后，`v` 子树还能不能绕过 `u` 连到上面。

如果 `low[v] == num[u]`，说明 `v` 子树只能回到 `u`，还是绕不过 `u`。

但现在删的是点 `u` 本身，所以这种情况下 `u` 依然是割点。

所以割点条件是：

~~~text
low[v] >= num[u]
~~~

---

## 10. 标准 DFS 伪代码

~~~cpp
dfs(u, parent):
    visited[u] = true
    num[u] = low[u] = dfsTime
    dfsTime++

    childCount = 0

    for each v in adj[u]:
        if not visited[v]:
            childCount++
            dfs(v, u)

            low[u] = min(low[u], low[v])

            // Bridge
            if low[v] > num[u]:
                (u, v) is a bridge

            // Articulation Point (non-root)
            if parent != -1 and low[v] >= num[u]:
                u is an articulation point

        else if v != parent:
            low[u] = min(low[u], num[v])

    // Articulation Point (root)
    if parent == -1 and childCount >= 2:
        u is an articulation point
~~~

## 11. 一个小例子

考虑这张图：

~~~text
0 - 1 - 2
    |
    3 - 4
~~~

### 哪些是割点？
- `1` 是割点：删掉后，`0`、`2`、`3-4` 会分开
- `3` 是割点：删掉后，`4` 会被断开

### 哪些是桥？
- `(0,1)`
- `(1,2)`
- `(1,3)`
- `(3,4)`

因为这张图本身就是一棵树，而**树中的每条边都是桥**。

---

## 12. 如果图里有环，会怎样？

如果某部分图形成一个环，比如：

~~~text
0 - 1 - 2
|       |
+-------+
~~~

那么这个环里的边通常都**不是桥**，因为删掉一条边仍然可以绕路。

同理：
- 环上的某些点也未必是割点
- 因为删掉它后，其他点仍可能通过环连通

所以：

> 环的存在会让 `low` 变小，从而避免某些点/边被判成关键结构。

---

## 13. 和记忆方法

### Bridge
记：

> 子树回不到我，父边就是桥

条件：

~~~text
low[v] > num[u]
~~~

---

### Articulation Point
记：

> 子树绕不过我，我就是割点

条件：

~~~text
low[v] >= num[u]
~~~

---

### 根节点
记：

> 根节点有两棵及以上 DFS 子树，就是割点

条件：

~~~text
rootChildren >= 2
~~~

---

## 14. 一句话总结

- `num[u]`：我什么时候第一次被访问
- `low[u]`：我这棵子树最早能绕回到哪里
- `low[v] > num[u]`：`(u,v)` 是桥
- `low[v] >= num[u]`：`u` 是割点（非根）
- 根节点有两个及以上子树：根是割点

---

## 15. 适用范围

这套判定方法适用于：

- **无向图**
- 求：
  - Articulation Points（割点）
  - Bridges（桥）

不直接用于有向图。  
有向图里一般讨论的是：
- SCC（Strongly Connected Components）
- Strong bridges / strong articulation points（更复杂）

---

## 16. 练习时最容易错的点

### 1. 忘记排除父边
无向图 DFS 时：

~~~text
else if v != parent
~~~

这一句很关键。

---

### 2. 桥和割点条件混淆
- 桥：`>`
- 割点：`>=`

---

### 3. 忘记根节点特判
根节点不能直接套普通割点公式，必须看：

~~~text
childCount >= 2
~~~

---

### 4. 误以为有环的地方一定没有割点
不一定。  
一个点可以一边连着环，一边连着树枝，仍可能是割点。

---

## 17. 面试 / 考试速记版

### 定义
- 割点：删点后连通分量增加
- 桥：删边后连通分量增加

### DFS 变量
- `num[u]`：访问次序
- `low[u]`：能回到的最早祖先

### 判定
- 桥：`low[v] > num[u]`
- 割点（非根）：`low[v] >= num[u]`
- 割点（根）：子树数 `>= 2`

### 本质
看子树能不能绕回祖先：
- 不能绕回 ⇒ 父边 / 父点关键
- 能绕回 ⇒ 不关键
